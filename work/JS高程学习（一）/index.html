< noscript >元素

Summary

JavaScript 是通过< script >元素插入到 HTML 页面中的。这个元素可用于把 JavaScript 代码嵌入到HTML 页面中，跟其他标记混合在一起，也可用于引入保存在外部文件中的 JavaScript。

  • 要包含外部 JavaScript 文件，必须将 src 属性设置为要包含文件的 URL。文件可以跟网页在同一台服务器上，也可以位于完全不同的域。
  • 所有< script >元素会依照它们在网页中出现的次序被解释。在不使用 defer 和 async 属性的情况下，包含在< script >元素中的代码必须严格按次序解释。
  • 对不推迟执行的脚本，浏览器必须解释完位于< script >元素中的代码，然后才能继续渲染页面的剩余部分。为此，通常应该把< script >元素放到页面末尾，介于主内容之后及< /body>标签之前。
  • 可以使用 defer 属性把脚本推迟到文档渲染完毕后再执行。推迟的脚本原则上按照它们被列出的次序执行。
  • 可以使用 async 属性表示脚本不需要等待其他脚本，同时也不阻塞文档渲染，即异步加载。异步脚本不能保证按照它们在页面中出现的次序执行。
  • 通过使用< noscript >元素，可以指定在浏览器不支持脚本时显示的内容。如果浏览器支持并启用脚本，则< noscript >元素中的任何内容都不会被渲染。

变量

var

声明后可以改变保存的值和类型，但不推荐

  • 声明作用域：为包含它的函数的局部变量，在函数退出时被销毁；如果去掉var声明，则可以创建一个全局变量。
  • 声明提升：会将声明提升到函数顶部，但赋值不会被提升，因此允许出现冗余声明。

let

  • 声明作用域：块级作用域，这是和var最大的区别，var是函数作用域。块作用域是函数作用域的子集，不允许冗余声明。
  • 暂时性死区：JS会注意到后面声明的let，但不提升，在声明之前不能以任何方式来引用未声明的变量。
  • 全局声明：全局作用域中let的声明不会成为window对象的属性，var会。
  • 条件声明：没看懂
  • for循环中的let：迭代变量的作用域仅限于for循环内部。var在退出循环时，迭代变量保存的时导致循环退出的值；let在后台为每个迭代循环声明一个新的迭代变量。

const

与let基本相同，唯一重要的区别是它声明变量时必须同时初始化变量且尝试修改该变量会导致错误。不允许重复声明，块级作用域。
